A) HTML Structure - top

Below is the HTML structure of the summary section, as an example. A simple 12 column text section with a title and a button.

		<!-- Begin Summary -->

B) CSS Files and Structure - top

I'm using 1 CSS file in this theme, which controls all of the styles besides royal_preloader.css (the image preloader untilized before you get to the theme main page) and ionicons.css (the icon fonts I used.) We of course also have the bootstrap css files. This is also not counting the onepage style.css files, which the only difference is some menu fixes.

The main CSS file, style.css is separated into sections using this format:

		/* =Table of Contents
		--------------------------------------------------------------
			=Common Styles
			=Typography
		*/

		/* =Common Styles
		-------------------------------------------------------------- */
		css code

		/* =Typography
		-------------------------------------------------------------- */
		css code


If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate class that needs to be edited. You can also use your browsers inspect element function to find the class that you need to customize.

C) JavaScript - top

No javascript files have to be customized for the theme to function, there is only the royal preloader script in the head tag which you will have to change to your own image paths. All animation and parallax is done in the HTML file with fairly simple code, which is explained in the section below.

  1. bootstrap.min.js - Bootstrap .js file
  2. jquery.easing.min.js - Required for portfolio sorting
  3. jquery.mixitup.min.js - Mixitup sorting plugin, equired for portfolio sorting - Documentation
  4. portfolio.js - Runs mixitup sorting plugin
  5. navbar.js - Smooth scroll links + a navbar fix
  6. ticker.js - Ticker in full screen hero section
  7. modal-fix.js - (no editing)
  8. skrollr.min.js - Skrollr Parallax + animation plugin - Documentation
  9. start-skrollr.js - Runs Skrollr
  10. scrollReveal.js - Animates elements into the viewport - Documentation
  11. royal_preloader.min.js - The image preloader

E) Header & Navbar - top

In the header of the site we have a couple of important items, first we have the favicon and apple touch icons. You can replace these with your own files but make sure you keep the names / directory the same: 

			<link href="favicon.ico" rel="shortcut icon" type="image/x-icon" />
			<link rel="apple-touch-icon" sizes="144x144" href="apple-touch-icon-144x144.png">
			<link rel="apple-touch-icon" sizes="114x114" href="apple-touch-icon-114x114.png">
			<link rel="apple-touch-icon" sizes="72x72" href="apple-touch-icon-72x72.png">
			<link rel="apple-touch-icon" href="apple-touch-icon.png">

We also have our fonts which are Montserrat and Open Sans, jquery 1.9.1 is loaded, our preloader files which are royal_preloader.css and royal_preloader.min.js, and the preloader script which you will have to change to your own image paths. Which looks like:

			<script type="text/javascript">

				// Change to your image paths
			    var images = {
			    	'slide1':     		'http://aliensix.com/bolt1.3/onepage-static/backgrounds/bg.jpg',
			    	'slide2':     		'http://aliensix.com/bolt1.3/onepage-static/img/backgrounds/bg2.jpg',
			    	'slide3':     		'http://aliensix.com/bolt1.3/onepage-static/img/backgrounds/bg3.jpg',
			    };
			 
			    // config
			    Royal_Preloader.config({
			        mode:       'text',
			        images:     images,
			        timeout:    10,
			        showInfo:   false,
			        background: ['#272727']
			    });

			</script>

Also our custom icon fonts called ionicons (cheatsheet here) are loaded below the preloader script. You also have the Bootstrap icon fonts available as well (cheatsheet here) which are already built in. In the <head> tag, you do not have to change anything but the title, image paths, favicons, and apple touch icons. This is the same on every page.

On the full site version each page has a parallax header, which can adjusted easily inside the HTML markup. Portfolio page header:

				<div class="row">

					<div class="col-lg-12 text-center">

					</div>

				</div>

Simply change: style="background-image:url('img/backgrounds/yourimage.jpg'); to your image, and adjust the h1 title and paragraph accordingly. You can align text on the left, right, or center. Using "text-left" "text-right" and "text-center" bootstrap classes.

After you are done setting up your pages you will need to match up the navigation on each page, simply copy the whole html code starting with <-- Begin Navigation --> and ending with <-- End Navigation --> and replace it on each page.

F) Portfolio - top

The portfolio section is very flexible. Columns can be adjusted easily if further customization is needed. The portfolio uses mixitup to sort the projects and the bootstrap modal feature to show more details on the project. You will have to update the individual project files and change 2 sets of images (a thumbnail and a fullsize version of the project), as well as change the markup in your portfolio html page (portfolio.html) for example, and change the filters to your own categories.

The project files are located in projects/project-1.html and so on. You can change the content for each project in the modal here. The modal is extremely flexible and can be adjusted to your needs easily.

You can keep the directories the same when you change the images to make it easier. The images are located in:

		img/projects
		img/projects/thumbs

The filters are labled "Portfolio Filters" in the HTML markup and can be changed accordingly there. You will just have to match up the filters: data-filter="web" with each portfolio item: li class="item web" the category is the class after item. The portfolio is very flexible and you can add as many portfolio items and filters as you need.

H) Parallax & Scroll Reveal - top

The animations fading the content into the screen as you scroll are controlled by scrollReveal.js and are also customized in the HTML, again extremely simple to customize as you can see in the 2nd screen capture. The code below controls how the column will fade into view, for example.

		<div data-scroll-reveal="enter top and move 50px over 1s" class="col-sm-4">

You can also just simplify it:

		<div data-scroll-reveal="enter top" class="col-sm-4">

The scrollReveal.js Documentation will cover a lot of these options, if you are looking to go above and beyond

Parallax features are controlled by skrollr.js and can be customized in the HTML easily. Skrollr also has other features that you can incorporate into this theme as well. Parallax can be added to sections easily by using the code below inside of the div element that has your background image.

The example below triggers the parallax image "img/backgrounds/bg.jpg" to parallax when the viewport is at the h1 tag inside of col-lg-12. Parallax can be added to anything and is done the same way as below, the skrollr.js Documentation will help you in customizing parallax further if needed.

		<div class="col-lg-12" 
		style="background-image:url('img/backgrounds/bg.jpg');padding:150px 0 150px 0;" 
		data-top-bottom="background-position: 50% 0px;" 
		data-bottom-top="background-position: 50% -200px;" 
		data-anchor-target=".col-lg-12 h1">

Your data anchor target should be the point of which you want the parallax to start. This tutorial uses the same method for parallax as this template as well if this seems confusing.
